# AltCrew — Pre-Launch SEO Audit

**Audit date:** 2026-04-27
**Mode:** Static source review (site not yet deployed at https://altcrew.in)
**Auditor scope:** Repo only — no live crawl, no CrUX/GSC data, no live SERP

---

## Executive Summary

**Overall SEO Health Score: 72 / 100**

Foundation is solid (clean metadata, multi-schema JSON-LD, llms.txt, single-page architecture suits the launch goal) but the site is **not ready to ship** as-is. The blocker is the missing Open Graph image — the single highest-leverage fix for a launch website where most discovery will be Instagram/WhatsApp/Twitter shares, not search.

### Top 5 critical / high

1. **No OG image** — every share to Instagram, WhatsApp, Twitter, LinkedIn renders a blank or fallback preview. Marketing kills you. **Critical.**
2. **No favicon / app icons** — no `favicon.ico`, `apple-touch-icon`, or web manifest. Tabs and home-screen adds will look unbranded. **High.**
3. **`metadataBase` and canonical point to `altcrew.in`** but domain registration / DNS not verified in repo. Verify it's live + has a 301 from `www`. **High.**
4. **Single-page architecture has zero internal landing pages** — losing potential rankings on city, sport, and event queries. Pre-launch trade-off; documented below. **Medium-but-strategic.**
5. **No `Person` schema for founder, no `BreadcrumbList`, no `LocalBusiness`** — schema coverage is thin given the launch event has a physical venue. **Medium.**

### Top 5 quick wins (under 1 hour each)

1. Generate 1200×630 OG image (hero composition + logo + tagline). Save to `public/og-default.jpg`. Reference in `metadata.openGraph.images` and `metadata.twitter.images`.
2. Add favicon + apple-touch-icon set. Next 16 supports `app/icon.tsx` for dynamic generation.
3. Add `Person` schema for the founder, linked from `orgSchema.founder`.
4. Add `LocalBusiness` (or `Place`) schema for the launch venue — Google indexes physical-location queries heavily for events.
5. Add `inLanguage`, `potentialAction` (search), and `WebPage` schema for the homepage.

## Content Quality — 18 / 23

### Strengths

- Strong, declarative voice. Hero uses concrete comparison ("Strava tells you what you did") — high E-E-A-T signal for AI citation.
- Founder note carries genuine experience (35 kg loss). Real-person Authority signal.
- llms.txt is well-structured and concise — directly citable by ChatGPT/Perplexity.

### Issues

- **Hero H1 brand comparison risk.** Using "Strava" is nominative fair use and legally fine, but Google may flag heavy reliance on a competitor brand name in the title slot. Mitigation: keep Strava in the H1 (good positioning) but ensure the `<title>` and `<meta description>` lead with AltCrew's own value prop (currently they do — good).
- **Description length.** Current `description` (252 chars) exceeds Google's typical 160-char SERP truncation. Trim to ~155 for primary, keep the long version for OG.
- **Thin "above-fold" copy on small screens.** Hero subhead is good. Verify the "find a crew to..." line doesn't get pushed below fold on 360px viewports — this is the page's promise.
- **No blog / content marketing.** Pre-launch, fine. Post-launch, plan a `/blog` for organic capture on long-tail queries (e.g., "best running clubs in Vizag").

## Performance (CWV) — Not measurable pre-deploy

Static-only review:

- **Geist font via `next/font`** — auto-subset, no CLS. ✅
- **Hero portraits = 4× ~800 KB JPGs.** Rendered at ~3:4 aspect, ~25vw on desktop, 45vw on mobile. **At ~800 KB each, these are 4× too large.** Recommended: re-encode at quality 75 with `sharp` to ~150-250 KB each. With four images priority-loading LCP, you'll hurt mobile LCP.
- **Founder image (`founder.jpeg`, 132 KB)** — fine size for a 56px circle, but consider a 128px source for better Retina rendering.
- **Motion (Framer Motion / `motion`) on every section** — high JS bundle cost. Verify the bundle size after build.
- **`next/image` used correctly** with `fill`, `sizes`, `priority` on first 2 hero tiles.

### Action

```bash
npx @squoosh/cli --mozjpeg '{"quality":78}' public/portraits/01-run.jpg
# repeat for 02-04. target: <250 KB per image
```

Or regenerate via Gemini at lower resolution, then `sharp` re-encode.
